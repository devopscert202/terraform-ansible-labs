String functions build predictable resource names. Lab 13 uses lower(replace(var.application, " ", "-")) to create a DNS-safe slug.

Example: Deduplicate CIDR list before creating subnets

Input: ["10.0.2.0/24", "10.0.1.0/24", "10.0.1.0/24"]

Expression: sort(tolist(toset(var.cidrs)))

Result: ["10.0.1.0/24", "10.0.2.0/24"] — duplicates removed, order stabilized.

2. Code examples from labs
variable "subnets" {
  type = map(object({ cidr = string, az = string }))
  default = {
    app_a = { cidr = "10.0.1.0/24", az = "us-east-1a" }
    app_b = { cidr = "10.0.2.0/24", az = "us-east-1b" }
  }
}

resource "terraform_data" "subnet" {
  for_each = var.subnets
  input    = { name = each.key, cidr = each.value.cidr, az = each.value.az }
}

output "zones" { value = sort(tolist(var.availability_zones)) }
locals {
  slug          = lower(replace(var.application, " ", "-"))
  unique_cidrs  = sort(tolist(toset(var.cidrs)))
  subnet_prefix = cidrsubnet("10.20.0.0/16", 8, 12)
  configuration = jsonencode({ name = local.slug, cidrs = local.unique_cidrs })
}
resource "aws_security_group" "service" {
  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      description = ingress.value.description
      from_port   = ingress.value.port
      to_port     = ingress.value.port
      protocol    = "tcp"
      cidr_blocks = ingress.value.cidr_blocks
    }
  }
}
# Prefer for_each with stable string keys:
resource "aws_subnet" "public" {
  for_each = var.public_subnets
  cidr_block = each.value.cidr
  tags = { Name = "${var.project}-${each.key}" }
}

# count is OK when index identity is intentional:
resource "example" "replica" {
  count = var.replica_count
}

5. Common pitfalls
  • Unstable keys: Using count with lists that reorder destroys/recreates resources.
  • Type coercion: toset requires all elements are known and unique-compatible.
  • Sensitive propagation: jsonencode of sensitive values marks output sensitive.
  • CIDR overlap: cidrsubnet math does not check for conflicts — validate separately.
  • Over-nesting: Deep nested comprehensions hurt readability — use locals.

aws_instance.web[*].id

# for expression — build map from another map
{ for k, v in var.subnets : k => v.cidr }

# for with filter
[ for k, v in var.subnets : k if can(v.public) && v.public ]

merge(var.tags, { Name = "${var.project}-${each.key}" })

12. FAQ

Functions vs resources?

Functions transform data at plan time; resources create infrastructure via providers.

Custom functions?

Not supported — compose locals with built-in functions instead.

Null handling?

Use coalesce(), try(), or nullable types with defaults in variables.
