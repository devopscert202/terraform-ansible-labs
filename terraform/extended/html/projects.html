3. Code tabs — capstone module
locals { name = "${var.project}-${terraform.workspace}" }

resource "aws_vpc" "this" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  tags                 = merge(var.tags, { Name = local.name })
}

resource "aws_subnet" "public" {
  for_each                = var.public_subnets
  vpc_id                  = aws_vpc.this.id
  cidr_block              = each.value.cidr
  availability_zone       = each.value.az
  map_public_ip_on_launch = true
  tags                    = merge(var.tags, { Name = "${local.name}-${each.key}" })
}
variable "public_subnets" {
  type = map(object({ cidr = string, az = string }))
  default = {
    public_a = { cidr = "10.50.1.0/24", az = "us-east-1a" }
    public_b = { cidr = "10.50.2.0/24", az = "us-east-1b" }
  }
}

variable "tags" {
  type    = map(string)
  default = { managed_by = "terraform", course = "extended" }
}
aws_region = "us-east-1"
project    = "capstone"
# Use AWS_PROFILE or IAM role — never access keys here

public_subnets = {
  public_a = { cidr = "10.50.1.0/24", az = "us-east-1a" }
  public_b = { cidr = "10.50.2.0/24", az = "us-east-1b" }
}
# Separate root module reads narrow outputs:
data "terraform_remote_state" "network" {
  backend = "s3"
  config = {
    bucket = "your-state-bucket"
    key    = "extended/dev/network/terraform.tfstate"
    region = "us-east-1"
  }
}

resource "aws_instance" "app" {
  subnet_id = values(data.terraform_remote_state.network.outputs.public_subnet_ids)[0]
}
4. Project design principles

Ownership boundary

This root module owns VPC and public subnets only. Shared services live in other state files.

Stable map keys

public_a and public_b keys survive reordering. Adding public_c does not recreate existing subnets.

Tagging standard

merge(var.tags, ...) applies platform tags consistently. Environment comes from workspace or tfvars.

Narrow outputs

Export vpc_id and subnet map — not every attribute. Consumers depend on contracts, not internals.

Provider pinning

required_providers block locks major versions. Run terraform init -upgrade deliberately.

Definition of done

fmt · init · validate · reviewed plan · apply · verify outputs · destroy training resources.
