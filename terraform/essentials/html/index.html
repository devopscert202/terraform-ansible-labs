Before You Start

Credential rule: Never put access_key or secret_key in Terraform source. Use AWS_PROFILE, environment credentials, or an IAM role.

Verify your setup: terraform version → export AWS_PROFILE=your-profile → aws sts get-caller-identity

Quick Start Commands
terraform version
# Expected: Terraform v1.5.x or later
export AWS_PROFILE=training
aws sts get-caller-identity
# Confirms account ID, user/role ARN — no keys in .tf files
cd terraform/essentials/labs/lab01-providers-init
terraform init
terraform validate
terraform plan
# random_pet.lab_id is created on apply — destroy when done
