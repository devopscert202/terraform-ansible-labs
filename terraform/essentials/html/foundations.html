Configuration drives everything

Your .tf files declare providers, resources, and outputs. Terraform parses them into an internal graph where edges represent dependencies (e.g., a subnet depends on its VPC).

Example: Lab 02 EC2 Deployment

Goal: A single Ubuntu EC2 instance tagged terraform-essentials-web in us-east-1.

Flow: data.aws_ami.ubuntu resolves the latest AMI → aws_instance.web references that ID → apply calls EC2 API → state stores i-0abc....

Auth: Provider uses AWS_PROFILE — never hard-code access keys.

2. Building Blocks (Click a Card)

terraform {} block

Core settings: required_version, required_providers, backend

provider {} block

Configures a plugin (region, aliases) — not credentials

resource {} block

Creates and manages a cloud object Terraform owns

data {} block

Read-only lookup of existing values (AMIs, VPCs)

output {} block

Exports values after apply (IDs, IPs, ARNs)

Version constraints

~> 5.0 allows 5.x patches, blocks 6.0

4. HCL Examples from the Labs
terraform {
  required_version = ">= 1.5.0"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
    random = {
      source  = "hashicorp/random"
      version = "~> 3.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

resource "random_pet" "lab_id" {
  length = 2
}

output "lab_id" {
  value = random_pet.lab_id.id
}
provider "aws" {
  region = var.aws_region
}

resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id
  instance_type = var.instance_type

  tags = {
    Name      = var.instance_name
    ManagedBy = "Terraform"
    Lab       = "terraform-essentials-lab02"
  }
}

output "instance_id" {
  value = aws_instance.web.id
}

output "public_ip" {
  value = aws_instance.web.public_ip
}
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"]  # Canonical

  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"]
  }

  filter {
    name   = "virtualization-type"
    values = ["hvm"]
  }
}
5. Provider Version Locking

After terraform init, Terraform writes .terraform.lock.hcl with exact provider checksums. Commit this file in team projects so everyone downloads the same binaries.

terraform init
→
.terraform.lock.hcl
→
.terraform/providers/

Common Beginner Mistakes

❌ Putting access_key / secret_key in a provider block

❌ Skipping terraform init after adding a new provider

❌ Editing terraform.tfstate by hand

✅ Use export AWS_PROFILE=... and run aws sts get-caller-identity first

6. Address Syntax

Every object has a unique address used in plans, state, and references:

resource "aws_instance" "web" { ... }
# Address: aws_instance.web
# Attributes: aws_instance.web.id, aws_instance.web.public_ip

data "aws_ami" "ubuntu" { ... }
# Address: data.aws_ami.ubuntu.id
