Terraform Workflow

Init, fmt, validate, plan, apply, and destroy — the daily operator loop.

Every Terraform change follows the same read → plan → apply rhythm. Labs 03–04 drill this until it becomes muscle memory.

Core Model

The Terraform Operator Loop

Write

Edit *.tf files describing desired infrastructure.

Format

terraform fmt normalizes HCL style — run before every commit.

Validate

terraform validate catches syntax and reference errors locally.

Plan

terraform plan shows create/update/destroy actions with diffs.

Review

Human or CI approves the plan output before any cloud mutation.

Apply

terraform apply executes the approved graph.

Destroy

terraform destroy tears down all managed resources in the root module.

Lab 03

Plan, Apply, Destroy Lifecycle

1

cd lab03

cd terraform/essentials/labs/lab03-plan-apply-destroy

2

init

terraform init if not already initialized.

3

plan

terraform plan -out=tfplan saves the plan to a file.

4

review

Read the plan: count adds/changes/destroys. Confirm instance type and AMI.

5

apply saved plan

terraform apply tfplan — applies exactly what was planned.

6

verify

terraform show or AWS console — confirm resource exists.

7

destroy

terraform destroy — remove all resources when lab is done.

Fmt and Validate Before Plan

Lab 04 — fmt and validate
cd terraform/essentials/labs/lab04-fmt-validate
terraform fmt -recursive
terraform validate
Check formatting diff
terraform fmt -check -recursive
# non-zero exit if files need formatting
CI-style sequence
terraform init -input=false
terraform fmt -check -recursive
terraform validate
terraform plan -input=false -out=tfplan
