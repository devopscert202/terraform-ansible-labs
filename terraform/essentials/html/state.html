3. Inspection Commands — Click Tabs
cd terraform/essentials/labs/lab06-local-state
terraform apply

terraform state list
# random_pet.first
terraform state show random_pet.first

# id          = "state-lab-happy-dog"
# length      = 2
# prefix      = "state-lab"
# ...
terraform show              # human-readable full state
terraform show -json        # automation / debugging
ls -la terraform.tfstate*
# terraform.tfstate         — current
# terraform.tfstate.backup  — previous version after mutation

cp terraform.tfstate terraform.tfstate.manual-backup

7. State Subcommands Reference
terraform state list
terraform state show 'random_pet.first'
terraform state mv 'old.name' 'new.name'
terraform state rm 'resource.name'    # removes from state only
terraform state pull > backup.tfstate # remote backends

Common Mistakes

❌ Running Terraform from repo root — wrong state file

❌ Committing state — leaks IDs and secrets

❌ Deleting state to fix errors — orphans real cloud resources

✅ One root module directory per state boundary

8. Lab 06 Configuration
resource "random_pet" "first" {
  prefix = "state-lab"
  length = 2
}

output "first_resource" {
  value = random_pet.first.id
