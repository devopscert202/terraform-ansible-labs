Variable precedence (highest wins)

Command-line -var overrides *.auto.tfvars, which overrides terraform.tfvars, which overrides TF_VAR_ environment variables, which overrides the default in the variable block.

Lab 05 Scenario: Tag Merge

Input: var.tags = { Environment = "training", Owner = "platform-team" }

Local: common_tags merges var.tags with Name, Service, ManagedBy

Result: EC2 instance gets all five tags from a single tags = local.common_tags line

3. HCL Examples (Code Tabs)
variable "instance_type" {
  description = "EC2 instance type."
  type        = string
  default     = "t3.micro"
}

variable "tags" {
  type = map(string)
  default = {
    Environment = "training"
    Owner       = "platform-team"
  }
}
locals {
  common_tags = merge(var.tags, {
    Name      = var.server_name
    Service   = "terraform-essentials"
    ManagedBy = "Terraform"
  })
}

resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id
  instance_type = var.instance_type
  tags          = local.common_tags
}
# lab05 — override defaults without editing .tf files
aws_region    = "us-east-1"
server_name   = "variables-lab-web"
instance_type = "t3.micro"

tags = {
  Environment = "training"
  Owner       = "platform-team"
}
variable "cloud" {
  type = string
  validation {
    condition     = contains(["aws", "azure", "gcp", "vmware"], var.cloud)
    error_message = "cloud must be aws, azure, gcp, or vmware."
  }
}

variable "cost_code" {
  type = string
  validation {
    condition     = length(var.cost_code) == 3
    error_message = "cost_code must contain exactly three characters."
  }
}
# Never commit real secrets to terraform.tfvars
export TF_VAR_phone_number="example-only-not-real"

4. Variable Precedence (Highest First)
1
-var and -var-file on CLI — explicit overrides for one-off runs
2
*.auto.tfvars — loaded automatically (alphabetical order)
3
terraform.tfvars — conventional environment-specific values
4
TF_VAR_name environment variables — ideal for secrets in CI
5
default in variable block — fallback when nothing else is set

Common Mistakes

❌ Committing terraform.tfvars with real API keys or passwords

❌ Using untyped variables when type constraints would catch errors early

❌ Thinking sensitive = true keeps values out of state (it does not)

✅ Use terraform.tfvars.example with placeholder values in git; real values via TF_VAR or secret manager
