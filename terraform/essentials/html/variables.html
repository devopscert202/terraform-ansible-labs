Variables and secrets

Inputs make configurations reusable; validation prevents unsafe or malformed values.

OverviewFoundationsWorkflowVariablesStateModules

Use typed inputs

variable "environment" {
  type = string
  validation {
    condition     = contains(["dev", "test", "prod"], var.environment)
    error_message = "Use dev, test, or prod."
  }
}

Use terraform.tfvars for non-secret local values. Pass secrets at runtime via a protected system such as CI secret variables or TF_VAR_name. Mark sensitive inputs and outputs, but protect state too: it can still store the underlying value.
