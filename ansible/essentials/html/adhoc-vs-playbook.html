Ad Hoc vs Playbook

Same modules — different packaging

Ad hoc command

  • One module, one shot
  • Great for probes and quick fixes
  • Not stored in Git by default
ansible web1 -i inventory/hosts.ini \
  -b -m ansible.builtin.apt \
  -a "name=tree state=present"

Playbook

  • Many tasks, version controlled
  • Handlers, roles, variables
  • Production automation
ansible-playbook -i inventory/hosts.ini \
  playbooks/apache.yml

When to use which

Ad hoc: ping, uptime, install one package to test sudo, gather a fact.

Playbook: Apache install, MOTD template, Node.js capstone—anything you will run again.
