Roles & Ansible Vault

Reusable automation packages + encrypted secrets at rest

Repository files: roles/webserver/ · playbooks/role-site.yml · vault/secrets.yml · playbooks/nodejs.yml
1. Role Directory Tree (click each item)
roles/webserver/
├── defaults/main.yml
├── tasks/main.yml
├── handlers/main.yml
├── templates/ (optional)

roles/webserver/ — Self-contained unit invoked with roles: [webserver] in playbook. Ansible auto-loads tasks, handlers, defaults in order.
defaultsLow-precedence variables. Easy to override.
tasksRequired. Main work happens here.
handlersNotified restarts and reloads.

3. Code Examples
# labs/playbooks/role-site.yml
---
- name: Site with webserver role
  hosts: webservers
  become: true
  roles:
    - webserver
# roles/webserver/tasks/main.yml
- name: Install web package
  ansible.builtin.apt:
    name: "{{ web_package }}"
    state: present
    update_cache: true

- name: Enable mod_rewrite
  ansible.builtin.apache2_module:
    name: rewrite
    state: present
  notify: Restart web service

- name: Ensure web service running
  ansible.builtin.service:
    name: "{{ web_service }}"
    state: started
    enabled: true
# Create password file (lab only)
echo 'lab-vault-password' > ~/.vault_pass && chmod 600 ~/.vault_pass

# Encrypt
ansible-vault encrypt vault/secrets.yml --vault-password-file ~/.vault_pass

# View without decrypting to disk
ansible-vault view vault/secrets.yml --vault-password-file ~/.vault_pass

# Edit in place
ansible-vault edit vault/secrets.yml --vault-password-file ~/.vault_pass

# Encrypt single variable
ansible-vault encrypt_string 'mysecret' --name api_token \
  --vault-password-file ~/.vault_pass
# labs/playbooks/nodejs.yml — Vault + Node.js 20
---
- name: Install Node.js 20.x
  hosts: webservers
  become: true
  vars_files:
    - ../vault/secrets.yml
  tasks:
    - name: Show token length (uses vault var)
      ansible.builtin.debug:
        msg: "api_token length is {{ api_token | length }}"

# Run with vault password
ansible-playbook -i inventory/hosts.ini.local playbooks/nodejs.yml \
  --vault-password-file ~/.vault_pass
# Create new role skeleton
ansible-galaxy init webserver

# Install role from Galaxy
ansible-galaxy install geerlingguy.apache

# requirements.yml for roles
---
roles:
  - name: geerlingguy.apache
    version: 3.1.0
4. Role vs Playbook — When to Extract
SituationRecommendation
Single-purpose lab playbookInline tasks OK (apache.yml)
Reused across playbooksExtract to role (webserver)
Sharing on GalaxyRole with meta/main.yml
Secrets in GitVault encrypt + vars_files
5. Variable Precedence in Roles
CLI extra vars (-e)           ← highest
play vars / vars_files
inventory group_vars / host_vars
role vars (vars/main.yml)
role defaults (defaults/main.yml)  ← lowest

Override web_package in group_vars/webservers.yml without editing roles/webserver/tasks/main.yml.
