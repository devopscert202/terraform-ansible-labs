Ansible Architecture

Agentless configuration over SSH (port 22)

Control node → managed nodes

Control node
ansible / ansible-playbook
SSH →
web1
web2
db1
Inventory — lists hosts and groups
Playbook — desired state in YAML
Modules — apt, service, template, …
No agent — Python + SSH on targets only

Typical workflow

  1. Edit inventory with private IPs and ansible_user
  2. Test: ansible webservers -m ansible.builtin.ping
  3. Apply: ansible-playbook playbooks/apache.yml
ansible -i inventory/hosts.ini webservers -m ansible.builtin.ping
