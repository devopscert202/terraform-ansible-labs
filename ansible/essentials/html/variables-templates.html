Repository files: inventory/group_vars/webservers.yml · templates/motd.j2 · playbooks/vars-demo.yml
1. Variable Precedence Ladder (click each level — lowest to highest wins)
1
Role defaultsroles/<role>/defaults/main.yml — lowest priority
↑ overrides below
2
Inventory group_vars / host_varsinventory/group_vars/webservers.yml
↑
3
Facts (setup module)ansible_distribution, ansible_hostname, etc.
↑
4
Play vars / vars_filesvars: block or vars_files in playbook
↑
5
Task vars / include_varsvars: on individual tasks
↑
6
Extra vars (-e)ansible-playbook site.yml -e "app_env=staging" — always wins
Play vars / vars_files: Defined in the playbook YAML. vars_files loads encrypted vault files. Higher than inventory group_vars.
2. Template Rendering Flow

webservers.yml
+
templates/
motd.j2
→
ansible.builtin
.template

/etc/motd

Template module renders Jinja2 on the control node, then copies result to target. Only writes file if content changed (idempotent).
3. Lab Example — Source and Output

group_vars/webservers.yml

webserver_port: 80
app_env: production

templates/motd.j2

Welcome to {{ inventory_hostname }}
Environment: {{ app_env }}
Web port: {{ webserver_port }}

Rendered on web1

Welcome to web1
Environment: production
Web port: 80

playbooks/vars-demo.yml

- name: Template motd
  ansible.builtin.template:
    src: ../templates/motd.j2
    dest: /etc/motd
    mode: "0644"

ansible webservers -i inventory/hosts.yaml \
  -m ansible.builtin.debug -a "var=webserver_port"

# In playbook
- name: Show app environment
  ansible.builtin.debug:

# Common Jinja2 in templates
{{ app_env | upper }}                    # PRODUCTION
{{ webserver_port | string }}          # "80"
{% if app_env == 'production' %}       # conditionals
  Strict mode enabled
{% endif %}
{{ groups['webservers'] | join(',') }} # web1,web2
{{ ansible_date_time.iso8601 }}        # timestamp from facts
# Capture task output in a variable
- name: Check apache
  ansible.builtin.command: systemctl is-active apache2
  register: apache_status
  changed_when: false

- name: Report status
  ansible.builtin.debug:
    msg: "Apache is {{ apache_status.stdout }}"
# Magic variables (always available)
{{ inventory_hostname }}   # web1
{{ group_names }}          # ['webservers']
{{ hostvars['web2']['ansible_host'] }}
{{ ansible_default_ipv4.address }}

7. Troubleshooting Variables
SymptomCauseFix
