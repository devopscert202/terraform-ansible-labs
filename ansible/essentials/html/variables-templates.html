Variables & Jinja2

Variable precedence, facts, register, and Jinja2 templates.

Variables flow from inventory, playbooks, and facts into Jinja2 templates and module arguments.

Core Model

Variable Sources

Inventory vars

group_vars, host_vars — lowest friction defaults.

Play vars

vars: block in play — play-scoped.

Task vars

module parameters and task-level vars.

Facts

ansible_* from setup — discovered host state.

register

Capture task output for later tasks.

Templates

.j2 files rendered to destination paths.

Template Rendering

Template task
- name: Deploy config
  ansible.builtin.template:
    src: app.conf.j2
    dest: /etc/app/app.conf
    mode: '0644'
Jinja2 in template

port = {{ http_port }}
