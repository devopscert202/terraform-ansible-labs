Lesson 02 · Inventory

Inventory & Groups Flow

How Ansible discovers hosts, groups, and variables before running any task

Repository files:
  • labs/inventory/hosts.ini — INI format with webservers and dbservers
  • labs/inventory/hosts.yaml — equivalent YAML inventory
  • labs/inventory/group_vars/webservers.yml — group-level variables
1. Inventory Resolution Flow
ansible / ansible-playbook
→
-i inventory/hosts.yaml
→
Parse INI or YAML
→
Load group_vars/
→
Match host pattern
→
SSH to each host
CLI entry: Every Ansible command needs an inventory source via -i or inventory = in ansible.cfg. The inventory plugin parses the file and builds an in-memory graph of groups and hosts.

3. Format Comparison — INI vs YAML

Use the group hierarchy above. Both INI and YAML express the same topology — pick one style per project and stay consistent.

FeatureINI (hosts.ini)YAML (hosts.yaml)
ReadabilityCompact for small inventoriesBetter for nested groups
Group vars[group:vars] sectionvars: under group
Host varsInline on host lineNested under host key
Dynamic inventorySeparate pluginSame plugins apply
 
# labs/inventory/hosts.ini
[webservers]
web1 ansible_host=10.0.1.10 ansible_user=ubuntu
web2 ansible_host=10.0.1.11 ansible_user=ubuntu

[webservers:vars]
ansible_python_interpreter=/usr/bin/python3

[dbservers]
db1 ansible_host=10.0.1.20 ansible_user=ubuntu
# labs/inventory/hosts.yaml
all:
  children:
    webservers:
      hosts:
        web1:
          ansible_host: 10.0.1.10
          ansible_user: ubuntu
        web2:
          ansible_host: 10.0.1.11
          ansible_user: ubuntu
      vars:
        ansible_python_interpreter: /usr/bin/python3
    dbservers:
      hosts:
        db1:
          ansible_host: 10.0.1.20
          ansible_user: ubuntu
# labs/inventory/group_vars/webservers.yml
---
webserver_port: 80
app_env: production

Ansible auto-discovers group_vars/<groupname>.yml adjacent to the inventory file. Directory layout: inventory/group_vars/webservers.yml applies to all hosts in webservers. 

# List inventory as graph
ansible-inventory -i inventory/hosts.yaml --graph

# Show all variables for a host (merged)
ansible-inventory -i inventory/hosts.yaml --host web1

# List hosts in a group
ansible webservers -i inventory/hosts.yaml --list-hosts

# Debug a group variable
ansible webservers -i inventory/hosts.yaml -m ansible.builtin.debug -a "var=webserver_port"
4. Host Patterns (click to learn)
allEvery host in inventoryansible all -m ping
webserversAll hosts in groupansible webservers -m ping
web1Single host by nameansible web1 -m ping
web1:web2Multiple hosts (OR)ansible web1:web2 -m ping
webservers:&dbserversIntersectionansible 'webservers:&dbservers'
all:!dbserversExclusionansible 'all:!dbservers'

6. Local Override Pattern (labs)
# Never commit real IPs — use a local overlay
cp inventory/hosts.ini.local.example inventory/hosts.ini.local
# Edit private IPs, then:
ansible -i inventory/hosts.ini.local webservers -m ansible.builtin.ping
7. Dynamic vs Static Inventory
TypeSourceWhen to use
Statichosts.ini / hosts.yaml fileLabs, small fleets, fixed IPs
Dynamicaws_ec2, azure_rm, gcp_compute pluginsCloud auto-scaling, ephemeral VMs
Inventory directoryinventory/ with multiple filesLarge orgs split by env/region

This essentials track uses static inventory only. Terraform outputs feed your hosts.ini.local private IPs.
