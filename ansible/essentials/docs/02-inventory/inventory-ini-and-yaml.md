# Ansible Inventory: INI and YAML

## Objective (conceptual)

**Inventory** is Ansible's source of truth for *which hosts exist* and *how to reach them*. Hosts sit in **groups** (`webservers`, `dbservers`); variables attach at host or group level. Ansible accepts **INI** and **YAML** formats—same information, different syntax.

The mental model: inventory is the **address book**; playbooks dial patterns like `webservers` instead of typing IPs each time.

**Interactive reference:** [Inventory and Groups](../../html/inventory-flow.html)

## INI format (Lab default)

`inventory/hosts.ini`:

```ini
[webservers]
web1 ansible_host=10.0.1.10 ansible_user=ubuntu
web2 ansible_host=10.0.1.11 ansible_user=ubuntu

[webservers:vars]
ansible_python_interpreter=/usr/bin/python3

[dbservers]
db1 ansible_host=10.0.1.20 ansible_user=ubuntu
```

- Group name in brackets `[webservers]`.
- `ansible_host` — connection address (may differ from inventory name).
- `ansible_user` — SSH user.
- `[webservers:vars]` — variables applied to every host in the group.

## YAML format (equivalent)

`inventory/hosts.yaml`:

```yaml
all:
  children:
    webservers:
      hosts:
        web1:
          ansible_host: 10.0.1.10
          ansible_user: ubuntu
        web2:
          ansible_host: 10.0.1.11
          ansible_user: ubuntu
      vars:
        ansible_python_interpreter: /usr/bin/python3
    dbservers:
      hosts:
        db1:
          ansible_host: 10.0.1.20
          ansible_user: ubuntu
```

Use YAML when inventory is generated by automation or nested groups grow deep.

## Group variables

`inventory/group_vars/webservers.yml`:

```yaml
---
webserver_port: 80
app_env: production
```

Ansible auto-loads `group_vars/<groupname>.yml` for matching groups.

## Host patterns in playbooks

| Pattern | Matches |
|---------|---------|
| `webservers` | All hosts in group |
| `web1` | Single host |
| `webservers:dbservers` | Union |
| `webservers:&dbservers` | Intersection (if any) |
| `*.example.com` | Wildcard (with inventory plugin support) |

## ansible.cfg inventory pointer

```ini
[defaults]
inventory = inventory/hosts.ini
```

Override per run: `ansible-playbook -i inventory/hosts.yaml site.yml`.

## Local overrides

`hosts.ini.local.example` documents personal IPs—copy to a gitignored file for your lab VMs without committing secrets or real addresses.

## Static vs dynamic (preview)

Essentials uses **static** files. Extended track adds `amazon.aws.aws_ec2` inventory plugin for live EC2 lists.

## Implicit localhost

`localhost` and `127.0.0.1` can run modules locally with `connection: local`—useful for API tasks on the control node. Essentials labs target remote `webservers` instead.

```bash
ansible localhost -m ansible.builtin.debug -a "msg=hello" -c local
```

## Operational commands (reference)

```bash
cd ansible/essentials/labs
ansible-inventory --list
ansible-inventory --graph
ansible webservers --list-hosts
ansible webservers -m ansible.builtin.setup -a "filter=ansible_distribution*"
```
