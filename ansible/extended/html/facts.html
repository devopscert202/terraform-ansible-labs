Facts & Custom Facts

Ansible discovers host state through the setup module. Facts become ansible_* variables you use in templates, conditionals, and ad hoc commands.

Repository files for this lab:
  • labs/inventory/hosts.ini — static inventory for webservers and appservers
  • labmanuals/lab02-facts.md — step-by-step with validation commands
  • docs/facts/gathering-facts.md and docs/facts/custom-facts.md — reference docs

Example scenario: OS-aware package install

Problem: You need nginx on Debian hosts but httpd on Red Hat — without hard-coding per host.

Solution: Facts provide ansible_facts.os_family. A when: clause selects the correct package manager task.

Flow: Play starts → setup gathers ansible_distribution → task uses ansible.builtin.apt when family is Debian.

4. Code Examples (click tabs)
# Gather all facts on one host (verbose JSON output)
ansible -i inventory/hosts.ini web1 -m ansible.builtin.setup

# Facts are also gathered automatically at play start unless gather_facts: false
# Filter by glob pattern — only distribution keys returned
ansible -i inventory/hosts.ini webservers \
  -m ansible.builtin.setup -a "filter=ansible_distribution*"

# Multiple patterns (comma-separated)
ansible -i inventory/hosts.ini web1 \
  -m ansible.builtin.setup -a "filter=ansible_distribution*,ansible_memtotal_mb"
# Gather only specific subsets (faster on large fleets)
- name: Minimal fact gather
  ansible.builtin.setup:
    gather_subset:
      - "!all"
      - distribution
      - network

# Common subsets: all, min, hardware, network, virtual, ohai, facter
---
- name: Fast play without facts
  hosts: all
  gather_facts: false   # Skip automatic setup at play start
  tasks:
    - ansible.builtin.debug:
        msg: "No ansible_distribution available unless you run setup manually"
# Deploy INI-style fact file (lab exercise)
- ansible.builtin.copy:
    dest: /etc/ansible/facts.d/lab.fact
    mode: "0755"
    content: |
      [lab]
      tier=web
      course=extended

# After deploy, re-gather:
- ansible.builtin.setup:

# Access: {{ ansible_local.lab.tier }}  →  "web"
---
- name: Use facts in conditionals
  hosts: webservers
  tasks:
    - name: Show OS info
      ansible.builtin.debug:
        msg: "{{ inventory_hostname }} runs {{ ansible_facts.distribution }} {{ ansible_facts.distribution_version }}"

    - name: Install nginx on Debian family
      ansible.builtin.apt:
        name: nginx
        state: present
      when: ansible_facts.os_family == "Debian"

6. gather_subset Strategies

Full gather (default)

All subsets collected. Best for small inventories or when you need complete host context. Adds 2–10 seconds per host.

Minimal + distribution

gather_subset: "!all,min,distribution" — Fast OS detection for conditional package tasks.

Network only

gather_subset: "!all,network" — IP addressing without full hardware scan.

Fact caching (production)

Configure fact_caching in ansible.cfg to avoid re-gathering unchanged hosts between plays.
