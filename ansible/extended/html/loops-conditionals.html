Loops & Conditionals

Repeat tasks with loop and branch with when. Together they eliminate copy-paste YAML and enable fact-driven automation.

Repository files:
  • labs/playbooks/loops-packages.yml — package and user loops (Lab 04)
  • labs/playbooks/conditionals-os.yml — OS and group conditionals (Lab 05)
  • docs/playbooks/loops-and-conditionals.md — reference guide
1. Loop Execution Flow
baseline_packages[] → item = htop → ansible.builtin.apt → item = vim → ansible.builtin.apt → … repeat

Each iteration runs the same task module with a different item value. Use loop_control.label for readable output.

2. Loop Types Comparison
Syntax Status Item variable Best for Example
loop: ✅ Current standard item Lists, dicts, registered results loop: "{{ packages }}"
loop_control: ✅ Recommended Custom label, index, pause Large lists, nested loops label: "{{ item.name }}"
with_items: ⚠️ Legacy item Old playbooks only Migrate to loop
with_dict: ⚠️ Legacy item.key, item.value Dict iteration Use loop: "{{ mydict | dict2items }}"
until: + retries ✅ Retry pattern Same task retried Wait for service, API readiness until: result.status == 200

Example scenario: Multi-user provisioning

Requirement: Create deploy and monitor users with different groups on all webservers.

Loop: lab_users list of dicts — each iteration passes item.name, item.groups to ansible.builtin.user.

Conditional: SSH key task runs only when ~/.ssh/id_ed25519.pub exists on control node.

4. Loop vs When — When to use which

Use loop when…

  • Same module, many similar items
  • Installing a list of packages
  • Creating multiple users or directories
  • Processing registered results
  • Iterating over inventory groups

Use when when…

  • Entire task should be skipped
  • OS-specific package managers
  • Feature flags (enable_firewall)
  • Host tier differences (web vs app)
  • Combining multiple conditions with AND
5. Code Examples
- name: Install baseline packages
  ansible.builtin.apt:
    name: "{{ item }}"
    state: present
    update_cache: true
  loop: "{{ baseline_packages }}"
  loop_control:
    label: "{{ item }}"
lab_users:
  - name: deploy
    groups: sudo
    shell: /bin/bash
  - name: monitor
    groups: adm
    shell: /bin/bash

- name: Create lab users
  ansible.builtin.user:
    name: "{{ item.name }}"
    groups: "{{ item.groups }}"
    shell: "{{ item.shell }}"
  loop: "{{ lab_users }}"
  loop_control:
    label: "{{ item.name }}"
- name: Install nginx on Debian family
  ansible.builtin.apt:
    name: nginx
    state: present
  when: ansible_facts.os_family == "Debian"
# Multiple conditions = AND (all must be true)
- name: Enable UFW allow SSH
  community.general.ufw:
    rule: allow
    port: "22"
    proto: tcp
  when:
    - enable_firewall | bool
    - ansible_facts.os_family == "Debian"
- name: Configure web tier only
  ansible.builtin.template:
    src: index.html.j2
    dest: /var/www/html/index.html
  when: inventory_hostname in groups['webservers']
- name: Authorize deploy SSH key
  ansible.builtin.authorized_key:
    user: deploy
    key: "{{ lookup('file', '~/.ssh/id_ed25519.pub') }}"
  when: lookup('file', '~/.ssh/id_ed25519.pub', errors='ignore') | length > 0
  # Loop not needed — single conditional task
