Dynamic Inventory — AWS EC2 Plugin

The amazon.aws.aws_ec2 inventory plugin queries the EC2 API at runtime. No manual IP updates when instances launch or terminate.

Repository files:
  • labs/inventory/aws_ec2.yml — plugin configuration with filters and keyed_groups
  • labs/inventory/hosts.ini — static fallback for local labs
  • labmanuals/lab07-dynamic-inventory.md — full lab with AWS tagging steps

2. Inventory Refresh Flow
1. Run command → 2. Load aws_ec2.yml → 3. EC2 API query → 4. Apply filters → 5. keyed_groups → 6. compose vars
Every ansible command re-fetches inventory unless caching is configured. Terraform-provisioned instances appear automatically after tagging.

Example scenario: Auto Scaling replacement

Event: ASG terminates unhealthy instance i-0abc and launches i-0xyz.

Static inventory: Requires manual edit of hosts.ini — risk of stale IPs.

Dynamic inventory: Next ansible-inventory --graph shows i-0xyz in role_webservers automatically.

4. Plugin Configuration (code tabs)
---
plugin: amazon.aws.aws_ec2
regions:
  - us-east-1
filters:
  instance-state-name: running
  tag:Environment: ansible-lab
keyed_groups:
  - key: tags.Role
    prefix: role
  - key: placement.region
    prefix: aws_region
hostnames:
  - private-ip-address
compose:
  ansible_host: private_ip_address
  ansible_user: "'ubuntu'"
export AWS_PROFILE=lab-terraform
export AWS_DEFAULT_REGION=us-east-1

ansible-inventory -i inventory/aws_ec2.yml --graph

# Expected output:
@role_webservers:
  |--10.0.1.10
@role_appservers:
  |--10.0.1.12
@aws_region_us_east_1:
  |--10.0.1.10
  |--10.0.1.12
ansible-inventory -i inventory/aws_ec2.yml --list | jq '._meta.hostvars'

# Shows per-host variables:
# ansible_host, ansible_user, instance_id, tags, placement, etc.
# Control node must reach private IPs (same VPC)
ansible -i inventory/aws_ec2.yml role_webservers -m ansible.builtin.ping

# SUCCESS => { "ping": "pong" }
ansible-playbook -i inventory/aws_ec2.yml \
  playbooks/loops-packages.yml --limit role_webservers
pip install boto3 botocore
ansible-galaxy collection install amazon.aws

aws sts get-caller-identity --profile lab-terraform

6. Static vs Dynamic Comparison
AspectStatic (hosts.ini)Dynamic (aws_ec2.yml)
IP updatesManual edit requiredAutomatic on each run
AWS credentialsNot neededIAM profile or keys required
Group assignmentExplicit in INI sectionsFrom tags via keyed_groups
Best forFixed lab VMs, air-gappedCloud fleets, ASG, Terraform
DebuggingSimple file editCheck tags, region, filters
7. Troubleshooting Checklist
  • Credentials: aws sts get-caller-identity --profile lab-terraform
  • Region: echo $AWS_DEFAULT_REGION matches plugin regions
  • Tags: Environment=ansible-lab and Role=webservers|appservers on instances
  • Collection: ansible-galaxy collection list amazon.aws
  • SSH: Control node in VPC can reach private IPs on port 22
  • Empty inventory: filters too restrictive — test with fewer filters
