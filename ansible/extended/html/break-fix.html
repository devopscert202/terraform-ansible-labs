3. Five Drills (expand each)

01YAML indentation error

▼

Broken file: break-fix/drill-01-broken-yaml.yml

ERROR! We were unable to read valid YAML: found unacceptable key...

Extra space before ansible.builtin.apt breaks task list indentation.

  tasks:
    - name: Install curl
     ansible.builtin.apt:    # ← wrong indent (5 spaces, need 6)
        name: curl
  tasks:
    - name: Install curl
      ansible.builtin.apt:   # aligned with name:
        name: curl
        state: present
Fix: Align module key with name: under the task dash. Run ansible-playbook --syntax-check first.
ansible-playbook -i inventory/hosts.ini break-fix/solutions/drill-01-fixed.yml

02Missing become for apt

▼

Broken file: break-fix/drill-02-missing-become.yml

E: Could not open lock file /var/lib/dpkg/lock - open (13: Permission denied)
Fix: Add become: true at play level or -b on ad hoc commands.
ansible-playbook -i inventory/hosts.ini break-fix/solutions/drill-02-fixed.yml
# PLAY RECAP: failed=0

03Handler name mismatch

▼

Broken file: break-fix/drill-03-handler-mismatch.yml

(no error — handler silently skipped)

notify: restart nginx ≠ handler name: Restart nginx — case and spacing matter.

Fix: Make notify string exactly match handler name. Look for missing RUNNING HANDLER in output.

04Non-FQCN module name

▼

Broken file: break-fix/drill-04-non-fqcn.yml

[DEPRECATION WARNING]: community.general ping has been deprecated... Use ansible.builtin.ping
Fix: Use ansible.builtin.ping instead of short ping.

05Wrong Python interpreter

▼

Broken file: break-fix/drill-05-broken-inventory.ini

/usr/bin/python: not found
Fix: Set ansible_python_interpreter=/usr/bin/python3 in inventory vars.
ansible -i break-fix/solutions/drill-05-fixed-inventory.ini webservers -m ansible.builtin.ping

On-call scenario: "Playbook worked yesterday"

Report: CI pipeline fails after Ubuntu AMI upgrade.

Investigation order: syntax-check → ping with -vvv → check ansible_python_interpreter → verify become → compare handler names.

Resolution: AMI removed python2 symlink — update inventory group_vars to python3.

7. Prevention Checklist
  • Run ansible-playbook --syntax-check before every apply
  • Use ansible-lint in CI for FQCN and handler patterns
  • Pin ansible_python_interpreter in group_vars/all.yml
  • Name handlers with consistent Title Case; copy notify string from handler name
  • Test with --limit on one host before fleet rollout
8. Time-Boxed Drill Flow (15 min each)

Set a timer. Reproduce error → read first fatal line → apply fix → verify failed=0. Do not peek at solutions until attempt 2.
